FUN vs PUNK

punk.fun has two launch styles. They run on the same auction engine, so the buying experience is identical. The difference is where the token comes from and what happens to the liquidity afterwards.

Coming soon

FUN is the launch style available today. PUNK is on the roadmap and runs on the same engine, but it isn't live yet. This page explains both so you understand the full picture — if you're launching now, you're launching FUN.

FUN launches

FUN is the fast path. You don't need a token beforehand — punk.fun creates one for you. It's a minimal, fixed-supply token with no owner and no admin controls, designed to be simple and predictable. This makes FUN ideal for community tokens and memecoins where speed and a clean, trustless token matter most.

FUN tokens must come from punk.fun's own token creator. You can't point a FUN launch at an outside token — that guarantee is part of what makes a FUN launch a FUN launch.

Pick FUN if

You're starting from scratch, want the quickest possible path from idea to live market, and don't need any special token features.

PUNK launches Coming soon

PUNK is the bring-your-own-token path. If your project already has a token — or you've created one elsewhere with features a minimal token doesn't have — a PUNK launch lets you run it through the same auction without recreating it.

PUNK places no restriction on where the token came from. Any standard token works, including one that happens to have been created by punk.fun's token creator.

Pick PUNK if

You already have a token you want to sell, or you need a token with custom behaviour that the built-in creator doesn't provide.

What's the same

Everything a buyer experiences is identical between the two:

  • The same falling-price Dutch auction.
  • The same tranche structure and price decay.
  • The same 50/50 split between the sale allocation and the liquidity reserve.
  • The same commit-now, claim-later flow.
  • The same "sells out to complete" finish line.

If you're buying, you don't need to care whether a launch is FUN or PUNK — the process is the same either way.

What's different

The only meaningful differences are at the two ends of the process:

  1. The startFUN creates the token for you; PUNK uses a token you supply.
  2. The endBoth end with a live trading pool, but the post-sale liquidity is handled along a route specific to each style. In both cases the resulting position is controlled by the protocol, not the launcher.
Status

FUN is the launch style you can use today. PUNK shares the same auction engine and is planned, but it isn't available yet — treat the PUNK details above as a preview of what's coming. Check current product status on punk.fun before planning a launch.
